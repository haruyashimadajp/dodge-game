## 2. 座標（ざひょう）の考え方 ※超重要

画面の位置は **x（よこ）** と **y（たて）** の数字で表します。
**算数のグラフと違って、y は下にいくほど大きく**なります。

```
 (0,0)────────────── x が大きい →
   │  ┌────────────────────┐
   │  │                    │
   y  │      ゲーム画面       │   ← 画面の広さは 1200 × 750
 大き │  (W=1200, H=750)    │
 い   │                    │
 ↓   └────────────────────┘
                        (1200, 750)
```

- `x = 0` … 左はし　／　`x = 1200` … 右はし
- `y = 0` … 上はし　／　`y = 750` … 下はし
- 画面のまん中は `x = 600, y = 375`

### 速度（vx, vy）の向き
弾は「速度」を持って動きます。単位は **1秒あたり何ピクセル進むか**。

- `vx`（よこの速さ）… **プラスで右** へ、**マイナスで左** へ
- `vy`（たての速さ）… **プラスで下** へ、**マイナスで上** へ

例：`vx = 200, vy = 0` → 1秒で右に200px進む
例：`vx = 0, vy = 150` → 1秒で下に150px進む（上から降ってくる弾）

## 4. よく触る変数の説明

「変数（へんすう）」＝ 数字や設定に名前をつけて保存したもの。
ここの数字を変えると、ゲームの手ざわりが変わります。

### 4-1. PHYS（物理パラメータ）＝ キャラの動きの感触

```js
const PHYS = {
  gravity: 2200,    // 重力。大きいほど早く落ちる（ジャンプが重くなる）
  moveSpeed: 260,   // 左右の最高速度
  accel: 2600,      // 地上で加速する強さ（大きいとキビキビ動く）
  airAccel: 1800,   // 空中で左右に動かせる強さ
  friction: 2400,   // 地上でとまる時の止まりやすさ（大きいとすぐ止まる）
  jumpVel: 760,     // ジャンプの初速。大きいほど高く跳ぶ
  jumpCut: 0.45,    // ボタンを早く離した時にどれだけ低く跳ぶか（小さいほど低い）
  coyote: 0.09,     // 足場を踏み外しても少しの間ジャンプできる猶予（秒）
  buffer: 0.10,     // 着地ちょっと前にジャンプ押しても成立する猶予（秒）
  maxFall: 1400,    // 落下速度の上限
};
```

**よく調整するのは `jumpVel`（跳ぶ高さ）と `moveSpeed`（移動の速さ）と `gravity`（重さ）** です。
「もっと高く跳びたい」→ `jumpVel` を増やす、という感じ。

### 4-2. platforms（足場）＝ 地面とジャンプ台

```js
const platforms = [
  { x: 0,   y: GROUND_Y, w: W,   h: GROUND_H, ground: true }, // ← 一番下の地面
  { x: 120, y: H - 150,  w: 150, h: 16 },  // 左の足場
  { x: 530, y: H - 150,  w: 150, h: 16 },  // 右の足場
  { x: 320, y: H - 250,  w: 160, h: 16 },  // 真ん中の高い足場
];
```

1つの `{ }` が足場1枚です。

- `x` … 左はしの位置　`y` … 上面の高さ
- `w` … 横はば　`h` … あつみ
- `ground: true` は「一番下の地面」だけにつける目印（消さないで）

**足場を増やす**には、`{ x:300, y:H-220, w:120, h:16 },` のような行を足すだけ。
**足場を消す**には、その行を消すだけ（地面の行は残す）。

> `H - 180` は「下から180px上」という意味。数字を大きくすると高い位置になります。

### 4-3. COL（色）

弾や地面の色。基本は `style.css` の `:root` で決めていますが、
弾の色などをコードから直接いじることもできます（通常は触らなくてOK）。

## 4.5. ゲーム内の設定メニュー（コードを書かずに変えられる）

ここまでの `PHYS` などは**コードを書き換える**調整でしたが、
ゲーム画面の**設定メニューからマウス/タップで変えられる**項目もあります。
変えた値はブラウザに**保存**され、次回も覚えています。

### 開き方
- **タイトル画面**（ゲーム開始前）… 設定がそのまま表示される
- **プレイ中** … `Esc` キー、または画面右上の「**II**」ボタンで一時停止すると出る

### 普通設定（ゲーム内容に影響しない）
- **音量** … ジャンプ音・被弾音の大きさ（0で無音）

### デバッグ設定（ゲーム内容に影響する）
折りたたみを開くと出てきます。

| 項目 | 内容 |
|------|------|
| **移動速度** | 左右の動きの速さ（`PHYS.moveSpeed` を上書き） |
| **ジャンプ力** | ジャンプの高さ（`PHYS.jumpVel` を上書き） |
| **残機** | ミスできる回数。当たると1減り、少し無敵になって復活。0でゲームオーバー |
| **弾の速さ** | 弾全体のスピード倍率（30%〜200%） |
| **移動の仕方** | **滑る / 滑らない** を切り替え（下で説明） |

### 「移動の仕方」＝ 滑る / 滑らない
主人公の左右移動のクセを切り替えられます。

- **滑る**（初期設定）… マリオのように、ジワッと加速して、キーを離すとスーッと止まる。
  慣性（勢い）があり、操作に「重み」が出ます。
  （内部的に `PHYS.accel`・`friction`・`airAccel` が効いている状態）
- **滑らない** … キーを押した瞬間に**常に一定の速度**で動き、離した瞬間に**ピタッと止まる**。
  慣性ゼロ。狙った位置でキビキビ止めたい弾幕避けに向いています。

> どちらが正解ということはありません。作りたい弾幕の難しさや操作感に合わせて選んでください。
> 「滑る」時の重さは `PHYS` の `accel`（地上の加速）・`airAccel`（空中）・`friction`（止まりやすさ）で
> さらに細かく調整できます。

### 操作方法（タイトル画面のみ）
- **パソコン操作 / スマホ操作** を選べます。
- **スマホ操作**にすると、プレイ中に画面へ**十字キー（上下左右）と JUMP ボタン**が出て、
  タップで遊べます（キーボードも同時に使えます）。
- **十字キーの位置（左 / 右）** … スマホ操作時に十字キーを左下・右下どちらに出すか選べます
  （右にすると JUMP ボタンが左に移動）。

## 5. 弾幕（だんまく）の作り方 ★メイン★

ここが自分で作る部分です。
**「○秒に、この弾幕を出す」を時間順に並べていく** 方式（タイムライン）です。
音楽に合わせて弾幕を出したいので、ビートやサビのタイミングに合わせて
時刻を書いていけば、そのまま譜面になります。

### 5-1. 仕組み（タイムライン）

`game.js` 下半分の `buildScript()` という関数の中に、

```js
burst(時刻, () => { ここで弾を出す });
```

という行を **好きなだけ** 並べます。`時刻` はスタートからの秒数。
その時刻になった瞬間に、中の命令が **1回だけ** 実行されます。

```js
function buildScript() {
  const cues = [];
  const burst = (t, fn) => cues.push({ t, fn });
  const cx = W / 2, cy = H / 3;        // よく使う「中心」をあらかじめ用意

  // ===== ここに自分の譜面を書く =====
  burst(1.0, () => { ... });   // 1.0秒で発動
  burst(2.5, () => { ... });   // 2.5秒で発動
  burst(2.5, () => { ... });   // 同じ時刻に複数並べてもOK

  return cues.sort((a, b) => a.t - b.t);  // ← この行は消さない（時間順に並べ直す）
}
```

- 行の **順番は気にしなくてOK**。最後の `sort` が自動で時間順にしてくれます。
- 同じ時刻に何個書いても全部発動します。
- ゲームをやり直すと、また 0 秒から譜面が流れます。

### 5-2. チャージ（警告→発射）＝ `delay`

今回いちばん大事な新機能です。
弾は **「召喚した瞬間に飛ぶ」のではなく、少し待ってから飛ばせます**。

- 弾を出すとき、最後に **`delay`（秒）** を指定できます。
- `delay` の間、弾はその場所に **うすい赤い輪（警告リング）** として表示され、
  **動かず・当たり判定もありません**（中央が少しずつ満ちていきます）。
- `delay` が終わると、本物の弾になって **発射** されます。

これがあると **時計回りの弾幕がとても作りやすくなります**。
「リング全部を一度に召喚して、1発ごとに `delay` を少しずつ増やす」だけで、
弾が円に沿って順番に発射され、**ぐるっと回って見える** からです（→ `spiral`）。

### 5-3. 弾は「ただのオブジェクト」、出すのは `spawn` だけ

いちばん大事なところ。**弾は `{ }` で書いたデータ**で、`spawn(...)` に渡すと画面に出ます。
覚える道のりはこれだけ:

```js
spawn({ x: 600, y: -10, vx: 0, vy: 200, r: 8 });
//       出る位置        速さ(右,下)    大きさ
```

`spawn` に渡せる値（ぜんぶ省略可。書かなければ下の初期値）:

| 名前 | 意味 | 省略すると |
|------|------|-----------|
| `x`, `y` | 出てくる位置 | 0, 0 |
| `vx`, `vy` | よこ/たての速さ（＋で右/下、−で左/上） | 0, 0 |
| `r` | 半径（大きいほどデカい弾。大小自由！） | 6 |
| `delay` | 何秒「警告リング」を出してから発射するか | 0（すぐ発射） |
| `move` | **動き方**（5-4）。書かなければ「まっすぐ」 | straight |

例：画面まん中から、0.8秒の警告のあと、まっすぐ下に降る半径10の弾
```js
spawn({ x: 600, y: 200, vy: 180, r: 10, delay: 0.8 });
```

#### まとめて出す道具（中身は全部 `spawn` を呼ぶだけ）

| 書き方 | 意味 |
|--------|------|
| `ring({ x, y, count, speed, r, delay, start })` | 円のまわりに等間隔で `count` 発、**同時に**発射 |
| `spiral({ x, y, count, speed, r, turns, gap, start, delay })` | **回って見える渦**（時計回りの本命）。下で詳しく |
| `spinShape({ x, y, count, size, spin, grow, vx, vy, r, start, delay })` | 中心のまわりに弾を並べて**まるごと回す**（リング・多角形）。中心も動かせる（5-4） |
| `aimVel(x, y, speed)` | (x,y)から**プレイヤーを狙う**速度 `{vx, vy}` を返す |
| `playerXY()` | 今のプレイヤーの中心 `{ x, y }` |
| `rand(min, max)` | min〜maxのランダムな数 |
| `W` / `H` | 画面の横幅(1200)・高さ(750) |

#### `spiral` の引数（時計回りの本命）

```js
spiral({ x, y, count, speed, r, turns, gap, start, delay })
```
- `count` … 何発　`speed` … 飛ぶ速さ　`r` … 半径
- `turns` … 何周ぶん回すか（1で1周、2で2周。マイナスで反時計回り）
- `gap` … 1発ごとの発射の間隔（秒）。小さいほど密な渦巻きになる
- `start` … 最初の角度（ラジアン）。回り始める向き
- `delay` … 1発目を撃つまでの準備時間（全体の警告）

角度は **大きくなるほど画面では時計回り**（`x=cos, y=sin`、yは下向き）。

### 5-4. ★弾の「動き方」を自分で作る（move）★

弾は「まっすぐ進む」だけでなく、**回転・波・加速・ホーミング**など好きな動きにできます。
やり方はかんたん。**`spawn` に `move` という関数を1つ渡すだけ。**

**「動き方」とは『毎フレーム、弾の位置をどう変えるか』を書いた関数** です。

```js
move(b, dt)
//   b  … 弾そのもの。b.x / b.y を書き換えると弾が動く。
//        好きな値を持たせてOK（b.vx, b.angle, b.cx … なんでも、spawn の { } に書くだけ）。
//        b.age = 発射してからの秒数（ゆれや時間変化に便利）。
//   dt … このフレームの経過秒。
```

`game.js` の一番シンプルなお手本が、初期設定の **`straight`** です（これを真似ればOK）:

```js
function straight(b, dt) {     // まっすぐ進む
  b.x += b.vx * dt;
  b.y += b.vy * dt;
}
```

#### その場で書く（いちばん手軽）

`spawn({ ... move(b, dt) { ここに動き } })` と、弾を出すところに直接書けます。
動きが使う値（下の例の落ちる速さなど）も同じ `{ }` の中。

```js
// ジグザグに落ちてくる弾
spawn({
  x: 600, y: -10, r: 8,
  move(b, dt) {
    b.y += 180 * dt;                              // 下に落ちる
    b.x += Math.sin(b.age * 6) * 120 * dt;        // 左右にゆれる（b.age を使う）
  },
});
```

#### 名前をつけて使い回す

同じ動きを何度も使うなら、関数にして `move:` に渡します（`straight` の隣に置けばOK）:

```js
function zigzag(b, dt) {
  b.y += b.fall * dt;                             // 落ちる速さは b.fall（下で渡す）
  const dir = Math.floor(b.age / 0.3) % 2 === 0 ? 1 : -1;  // 0.3秒ごとに左右
  b.x += dir * b.sway * dt;
}

// 使うとき：動きが使う値（fall, sway）も { } に書けば、そのまま b.◯◯ で読める
spawn({ x: 600, y: -10, r: 8, fall: 180, sway: 220, move: zigzag });
```

> コツ：`b.age`（経過秒）と `dt`（フレーム秒）を組み合わせれば、たいていの動きが作れます。
> プレイヤーを狙いたいときは関数の中で `playerXY()` を呼べます。

#### `spinShape`（回転する図形のショートカット）

「中心のまわりに弾を並べて、まるごと回す」道具です（中身は `spawn` の `move` で回しているだけ）。
**`count` を多くすればリング、少なくすれば多角形**（4で正方形・3で三角形）。

```js
// 回り続けるリング
spinShape({ x: cx, y: cy, count: 14, size: 40, spin: 2.2, grow: 95, delay: 0.6 });

// 回りながら下へ飛んでくる正方形
spinShape({ x: cx, y: -40, count: 4, size: 42, spin: 2.6, vy: 230, delay: 0.5 });
```
- `count` … 弾の数（＝角の数）。多いほど円に近づく
- `size` … 中心からの距離（図形の大きさ）。`0` にして `grow` で広げてもOK
- `spin` … 回る速さ（＋で時計回り／−で反時計回り）
- `grow` … 1秒で `size` がどれだけ増えるか（0なら大きさ一定、＋で外へ広がる）
- `vx, vy` … 中心が動く速さ（0ならその場で回るだけ。＋vy で下へ飛ぶ）
- `r` … 弾の半径　`start` … 並べ始める角度（図形の傾き）　`delay` … 警告（準備）時間

## 6. 弾幕レシピ集（コピペで使える）

下の例は、`buildScript()` の中（`const cx ...` の下）に貼り付ければ動きます。
**今入っているサンプル（`burst(...)` 行）を消してから**貼ると分かりやすいです。

### レシピ① 警告つきの狙い撃ち
```js
burst(1.0, () => {
  const v = aimVel(cx, -20, 240);                     // プレイヤー方向へ
  spawn({ x: cx, y: -20, vx: v.vx, vy: v.vy, r: 14, delay: 0.8 });  // 0.8秒警告
});
```

### レシピ② 全方位リング（同時にバッと広がる）
```js
burst(5.0, () => ring({ x: cx, y: cy, count: 20, speed: 170, r: 7, delay: 1.0 }));
//               中心(cx,cy)から 20発 速さ170 半径7 警告1.0秒
```

### レシピ③ 時計回りの渦巻き（★delayの本領）
```js
burst(8.0, () => spiral({
  x: cx, y: cy, count: 60, speed: 190, r: 6,
  turns: 1,     // 1周ぶん
  gap: 0.04,    // 1発ごとに0.04秒ずらす → 順番に発射されて回って見える
  start: 0,     // 右からスタート
  delay: 0.5,   // 全体の準備（警告）0.5秒
}));
```
反時計回りにしたいときは `turns: -1` に。

### レシピ④ 上から雨のように降らせる（時刻を並べる）
```js
for (let t = 2; t < 12; t += 0.4) {       // 2秒から12秒まで0.4秒おき
  burst(t, () => {
    const r = rand(5, 16);                // 大きさバラバラの円
    spawn({ x: rand(0, W), y: -r, vy: rand(150, 230), r });
  });
}
```
> `for` で `burst(...)` をまとめて並べると、繰り返しっぽい弾幕も時刻指定で作れます。

### レシピ⑤ 左右の壁から飛んでくる
```js
for (let t = 3; t < 10; t += 0.45) {
  burst(t, () => spawn({ x: -10,    y: rand(0, H), vx:  200, r: rand(6, 12) })); // 左→右
  burst(t, () => spawn({ x: W + 10, y: rand(0, H), vx: -200, r: rand(6, 12) })); // 右→左
}
```

### レシピ⑥ 同時に複数のリングを時間差で
```js
burst(10.0, () => ring({ x: cx, y: cy, count: 24, speed: 160, r: 6, delay: 0.8 }));            // まず外向き
burst(10.0, () => ring({ x: cx, y: cy, count: 24, speed: 160, r: 6, delay: 1.6, start: Math.PI/24 })); // 少し回して2枚目
```

### レシピ⑦ 本当に回転する弾（spinShape）
```js
// 回り続けるリング
burst(12.0, () => spinShape({
  x: cx, y: cy, count: 14,
  size: 40,     // 中心からの距離
  spin: 2.2,    // 回る速さ（＋で時計回り）
  grow: 95,     // だんだん外へ広がる（0なら大きさ一定）
  delay: 0.6,   // 警告0.6秒
}));

// 回りながら降ってくる正方形（count を 4 に、中心を vy で動かすだけ）
burst(16.0, () => spinShape({ x: cx, y: -40, count: 4, size: 42, spin: 2.6, vy: 230, delay: 0.5 }));
```

### レシピ⑧ 自作の動きを使う（落ちながら左右にゆれる）
動きは `spawn` の中に直接書けます（5-4参照）:
```js
for (let t = 4; t < 14; t += 0.5) {
  const x = rand(100, W - 100);
  burst(t, () => spawn({
    x, y: -8, r: 8,
    move(b, dt) {
      b.y += 170 * dt;                        // 落ちる
      b.x = x + Math.sin(b.age * 6) * 60;     // 左右にゆれる
    },
  }));
}
```

## 7. つまづきポイント / FAQ

**Q. 変更したのに反映されない**
→ 保存（⌘S）した？ ブラウザをリロード（⌘R）した？ の2つを確認。

**Q. 画面が真っ暗・弾が出ない**
→ コードのどこかで打ち間違いの可能性。ブラウザで **右クリック→「検証」→「Console」タブ** を見ると、
赤いエラーメッセージが出ています。だいたい「○行目」と教えてくれます。
直前に変えた所を見直しましょう。

**Q. `return cues.sort(...)` を消してしまった**
→ `buildScript()` の **一番最後の行** に必ず必要です。これが無いと弾幕が出ません。
`burst(...)` の行は **この `return` より上** に書いてください。

**Q. カッコの対応がわからなくなった**
→ `(` `)`、`{` `}`、`[` `]` は必ずペアです。1つ消し忘れるとエラーになります。
エディタ（VS Code/Zed）はカッコを色で対応表示してくれるので分かりやすいです。

**Q. 警告（うすい輪）の時間を変えたい／無くしたい**
→ `spawn({ ... delay: 秒 })` の `delay` を変える。`0`（または省略）で警告なしの即発射です。

**Q. 弾を大きく/小さくしたい**
→ `spawn` の `r`（半径）を変える。`r: rand(5, 16)` のようにすると大小バラバラに。

**Q. 譜面が終わったあと弾が出ない**
→ 仕様です。`burst(...)` を書いた時刻ぶんだけ流れて終わります。長くしたいなら
あとの時刻に `burst(...)` を足してください（音楽の長さに合わせる想定）。
