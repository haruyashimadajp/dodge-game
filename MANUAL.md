## 2. 座標（ざひょう）の考え方 ※超重要

画面の位置は **x（よこ）** と **y（たて）** の数字で表します。
**算数のグラフと違って、y は下にいくほど大きく**なります。

```
 (0,0)────────────── x が大きい →
   │  ┌────────────────────┐
   │  │                    │
   y  │      ゲーム画面       │   ← 画面の広さは 960 × 600
 大き │   (W=960, H=600)    │
 い   │                    │
 ↓   └────────────────────┘
                          (960, 600)
```

- `x = 0` … 左はし　／　`x = 960` … 右はし
- `y = 0` … 上はし　／　`y = 600` … 下はし
- 画面のまん中は `x = 480, y = 300`

### 速度（vx, vy）の向き
弾は「速度」を持って動きます。単位は **1秒あたり何ピクセル進むか**。

- `vx`（よこの速さ）… **プラスで右** へ、**マイナスで左** へ
- `vy`（たての速さ）… **プラスで下** へ、**マイナスで上** へ

例：`vx = 200, vy = 0` → 1秒で右に200px進む
例：`vx = 0, vy = 150` → 1秒で下に150px進む（上から降ってくる弾）

## 4. よく触る変数の説明

「変数（へんすう）」＝ 数字や設定に名前をつけて保存したもの。
ここの数字を変えると、ゲームの手ざわりが変わります。

### 4-1. PHYS（物理パラメータ）＝ キャラの動きの感触

```js
const PHYS = {
  gravity: 2200,    // 重力。大きいほど早く落ちる（ジャンプが重くなる）
  moveSpeed: 260,   // 左右の最高速度
  accel: 2600,      // 地上で加速する強さ（大きいとキビキビ動く）
  airAccel: 1800,   // 空中で左右に動かせる強さ
  friction: 2400,   // 地上でとまる時の止まりやすさ（大きいとすぐ止まる）
  jumpVel: 760,     // ジャンプの初速。大きいほど高く跳ぶ
  jumpCut: 0.45,    // ボタンを早く離した時にどれだけ低く跳ぶか（小さいほど低い）
  coyote: 0.09,     // 足場を踏み外しても少しの間ジャンプできる猶予（秒）
  buffer: 0.10,     // 着地ちょっと前にジャンプ押しても成立する猶予（秒）
  maxFall: 1400,    // 落下速度の上限
};
```

**よく調整するのは `jumpVel`（跳ぶ高さ）と `moveSpeed`（移動の速さ）と `gravity`（重さ）** です。
「もっと高く跳びたい」→ `jumpVel` を増やす、という感じ。

### 4-2. platforms（足場）＝ 地面とジャンプ台

```js
const platforms = [
  { x: 0,   y: GROUND_Y, w: W,   h: GROUND_H, ground: true }, // ← 一番下の地面
  { x: 160, y: H - 180,  w: 180, h: 16 },  // 左の足場
  { x: 620, y: H - 180,  w: 180, h: 16 },  // 右の足場
  { x: 390, y: H - 300,  w: 180, h: 16 },  // 真ん中の高い足場
];
```

1つの `{ }` が足場1枚です。

- `x` … 左はしの位置　`y` … 上面の高さ
- `w` … 横はば　`h` … あつみ
- `ground: true` は「一番下の地面」だけにつける目印（消さないで）

**足場を増やす**には、`{ x:300, y:H-220, w:120, h:16 },` のような行を足すだけ。
**足場を消す**には、その行を消すだけ（地面の行は残す）。

> `H - 180` は「下から180px上」という意味。数字を大きくすると高い位置になります。

### 4-3. COL（色）

弾や地面の色。基本は `style.css` の `:root` で決めていますが、
弾の色などをコードから直接いじることもできます（通常は触らなくてOK）。

## 5. 弾幕（だんまく）の作り方 ★メイン★

ここが自分で作る部分です。

### 5-1. 仕組み

ゲーム中は毎フレーム（1秒に約60回）、`Patterns.tick(t, dt)` という関数が呼ばれます。
**この中に「弾を出す命令」を書く**と、弾が出ます。

```js
tick(t, dt) {
  this._dt = dt;

  // ここに弾を出すコードを書く！
}
```

- `t` … スタートしてから何秒たったか（例：12.5）。難易度を時間で変えるのに使う
- `dt` … 前のフレームからの経過秒（タイマー処理で内部的に使う）

### 5-2. 弾を出す命令たち（B.〇〇）

弾を出すための便利な道具が `B` にまとまっています。

| 書き方 | 意味 |
|--------|------|
| `B.spawn(x, y, vx, vy, r)` | 位置(x,y)に、速度(vx,vy)で動く、半径rの弾を出す |
| `B.aim(x, y, speed)` | (x,y)から**プレイヤーを狙う**速度を計算して返す |
| `B.player` | 今のプレイヤーの中心 `{ x, y }` |
| `B.W` / `B.H` | 画面の横幅(960)・高さ(600) |
| `B.rand(min, max)` | min〜maxのランダムな数（バラつきを出す時に使う） |
| `B.every(key, 秒, 関数)` | 〇秒ごとに1回だけ関数を実行する（弾を定期的に出す要） |

#### B.spawn の引数（一番大事）

```js
B.spawn(x, y, vx, vy, r)
//       │  │  │   │   └ r  : 半径（大きいほどデカい弾）大小自由！
//       │  │  │   └──── vy : たての速さ（＋で下、−で上）
//       │  │  └──────── vx : よこの速さ（＋で右、−で左）
//       │  └─────────── y  : 出てくる位置のたて
//       └────────────── x  : 出てくる位置のよこ
```

例：画面上のまん中から、まっすぐ下に降る半径10の弾
```js
B.spawn(480, -10, 0, 180, 10);
//      x=480(まん中), y=-10(画面の少し上から), vx=0, vy=180(下へ), r=10
```
> `y = -10` のように画面の外から出すと、スッと入ってくる自然な弾になります。

#### B.every の使い方（定期的に弾を出す）

`tick` は毎フレーム呼ばれるので、そこに `B.spawn` をそのまま書くと
**1秒に60発**出てしまいます。そこで `B.every` で間隔をあけます。

```js
B.every('rain', 0.5, () => {
  B.spawn(B.rand(0, B.W), -10, 0, 180, 8);
});
// → 0.5秒ごとに、上のどこかからランダムに弾を1発落とす
```

- 第1引数 `'rain'` は **名前（key）**。`B.every` を複数使うときは
  **それぞれ違う名前**にしてください（同じ名前だとタイマーが混ざります）。
- 第2引数 `0.5` は **間隔（秒）**。小さくすると弾が増えます。
- 第3引数 `() => { ... }` が **やること**（中で `B.spawn` する）。

## 6. 弾幕レシピ集（コピペで使える）

下の例は、`tick` の中（`this._dt = dt;` の下）に貼り付ければ動きます。
**今入っているサンプル（PLACEHOLDER の部分）を消してから**貼ってください。
いくつか組み合わせてもOKです。

### レシピ① 上から雨のように降らせる
```js
B.every('rain', 0.4, () => {
  const x = B.rand(0, B.W);
  const r = B.rand(5, 16);          // 大きさバラバラの円
  B.spawn(x, -r, 0, B.rand(120, 220), r);
});
```

### レシピ② プレイヤーを狙い撃ち
```js
B.every('snipe', 1.0, () => {
  const x = B.rand(0, B.W);
  const v = B.aim(x, -10, 260);     // プレイヤー方向の速度を計算
  B.spawn(x, -10, v.vx, v.vy, 10);
});
```

### レシピ③ 扇（おうぎ）状にばらまく
```js
B.every('fan', 1.2, () => {
  const cx = B.W / 2, cy = -10;     // 上のまん中から
  const n = 7;                      // 弾の数
  const spread = Math.PI / 4;       // 広がる角度
  const base = Math.PI / 2;         // 基準＝真下
  const speed = 180;
  for (let i = 0; i < n; i++) {
    const a = base - spread / 2 + spread * (i / (n - 1));
    B.spawn(cx, cy, Math.cos(a) * speed, Math.sin(a) * speed, 8);
  }
});
```

### レシピ④ 全方位リング（円形にバッと広がる）
```js
B.every('ring', 1.5, () => {
  const cx = B.W / 2, cy = B.H / 3; // 中心の位置
  const n = 16, speed = 150;        // 16方向に発射
  for (let i = 0; i < n; i++) {
    const a = (Math.PI * 2) * i / n;
    B.spawn(cx, cy, Math.cos(a) * speed, Math.sin(a) * speed, 7);
  }
});
```

### レシピ⑤ 左右の壁から飛んでくる
```js
B.every('wallL', 0.45, () => {            // 左から右へ
  B.spawn(-10, B.rand(0, B.H), 200, 0, B.rand(6, 12));
});
B.every('wallR', 0.45, () => {            // 右から左へ
  B.spawn(B.W + 10, B.rand(0, B.H), -200, 0, B.rand(6, 12));
});
```

### レシピ⑥ 時間で難易度を変える（段階的）
```js
if (t < 10) {
  // 0〜10秒：かんたん
  B.every('a', 0.6, () => B.spawn(B.rand(0, B.W), -10, 0, 150, 8));
} else if (t < 20) {
  // 10〜20秒：ふつう（雨＋狙い撃ち）
  B.every('a', 0.35, () => B.spawn(B.rand(0, B.W), -10, 0, 180, 8));
  B.every('b', 1.2,  () => { const v = B.aim(B.W/2, -10, 240); B.spawn(B.W/2, -10, v.vx, v.vy, 11); });
} else {
  // 20秒〜：むずかしい（リング追加）
  B.every('a', 0.25, () => B.spawn(B.rand(0, B.W), -10, 0, 220, 8));
  B.every('ring', 1.5, () => {
    for (let i = 0; i < 16; i++) {
      const ang = Math.PI * 2 * i / 16;
      B.spawn(B.W/2, B.H/3, Math.cos(ang)*150, Math.sin(ang)*150, 7);
    }
  });
}
```

## 7. つまづきポイント / FAQ

**Q. 変更したのに反映されない**
→ 保存（⌘S）した？ ブラウザをリロード（⌘R）した？ の2つを確認。

**Q. 画面が真っ暗・弾が出ない**
→ コードのどこかで打ち間違いの可能性。ブラウザで **右クリック→「検証」→「Console」タブ** を見ると、
赤いエラーメッセージが出ています。だいたい「○行目」と教えてくれます。
直前に変えた所を見直しましょう。

**Q. カッコの対応がわからなくなった**
→ `(` `)`、`{` `}`、`[` `]` は必ずペアです。1つ消し忘れるとエラーになります。
エディタ（VS Code/Zed）はカッコを色で対応表示してくれるので分かりやすいです。

**Q. 弾が多すぎて重い／少なすぎる**
→ `B.every` の秒数（間隔）を調整。大きく＝少なく、小さく＝多く。

**Q. 弾を大きく/小さくしたい**
→ `B.spawn` の最後の数字 `r`（半径）を変える。`B.rand(5,16)` のようにすると大小バラバラに。

**Q. 元のサンプル弾幕に戻したい**
→ `MANUAL.md` のレシピや、Gitで管理していれば前の状態に戻せます。
心配なら、書き換える前の `tick` の中身をメモ帳などにコピーしておくと安心です。
